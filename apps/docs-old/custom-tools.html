Core Concept

Custom Tools

Extend the Agent with your own domain-specific superpowers.

Overview

SynapseJS ships with 20 built-in tools, but you can register any number of additional custom tools. A tool definition consists of a name, description (shown to the LLM), a JSON Schema for its parameters, and a signal type that your frontend will receive and handle.

Full Example

lib/custom-tools.ts
import { ToolRegistry } from '@synapsejs/core';

// 1. Register the tool on the server (before Agent is used)
ToolRegistry.register({
  name: 'addToCart',
  description: 'Add a product to the shopping cart by its ID',
  parameters: {
    type: 'object',
    properties: {
      productId: { type: 'string', description: 'The product ID' },
      quantity: { type: 'number', description: 'Number of items' },
    },
    required: ['productId'],
  },
  signal: 'ADD_TO_CART',
});
app/page.tsx — handle the signal
const { processSignals } = useSynapseSignals({
  // 2. Handle your custom signal in the frontend
  ADD_TO_CART: ({ productId, quantity = 1 }) => {
    cartStore.add(productId, quantity);
    toast.success(`Added ${quantity} item(s) to cart!`);
  },
});

Tool Definition Schema

Field Type Description
namestringCamel-case function name the LLM calls.
descriptionstringShown directly to the LLM — be precise about when/why to use it.
parametersJSON Schema objectStandard JSON Schema for the tool's input parameters.
signalstringUPPER_SNAKE_CASE string that identifies this tool call on the frontend.
